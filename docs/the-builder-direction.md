# The Builder direction

Milestone 33 updates The Builder direction to use the first real Builder artwork now available for NozzleNote by BMR. This document remains internal direction only; it does not add public downloads, active paid flows, account access, checkout, license activation or image-generation prompts.

## Who The Builder represents

The Builder represents Pedro Luiz BMR in a stylized way. He is the founder-led visual character for NozzleNote by BMR and should support the build-in-public story behind a focused 3D printer maintenance logbook currently in development.

The Builder should communicate that NozzleNote is being built by a real independent product builder who is working carefully, practically and responsibly.

## Current available asset

The current real Builder asset is:

- `/assets/about/builder-hero.png`

The public About page currently uses only this hero Builder image. The image should be presented as a polished founder-character visual, with semantic alt text and responsive treatment that keeps the Premium Violet Tech direction: dark navy/black surfaces, cyan/blue primary accents, controlled violet highlights, subtle technical glow, clean cards and readable typography.

If the hero file is missing, the public page should fall back to a refined Builder visual treatment rather than a broken image icon or unfinished developer copy.

## Future optional assets

Additional Builder images are optional future assets, not requirements for the current public About page. They may be added later only when real artwork exists and can be integrated safely without making the page look unfinished.

Potential future designer-created artwork may use these pose directions:

1. Reviewing scattered maintenance notes, spreadsheets and printer-care reminders in a premium dark workspace.
2. Focused independent product builder at a workstation, working on NozzleNote interface concepts in a dark technical environment.
3. Reviewing product interface concepts without implying final screenshots, slicer controls, printer controls or manufacturer dashboards.
4. Looking at a roadmap or planning board for release preparation without implying active early access, active crowdfunding or available downloads.

Potential future stable paths, if real artwork is produced later:

- `/assets/about/builder-messy-records.png`
- `/assets/about/builder-at-work.png`
- `/assets/about/builder-product-interface.png`
- `/assets/about/builder-roadmap-cta.png`

These paths should not appear as visible public placeholder blocks unless final artwork exists and the page can render it intentionally.

## How The Builder should feel

The Builder should feel like a modern independent product builder:

- professional;
- calm and focused;
- technical but accessible;
- maker-friendly;
- premium;
- trustworthy;
- hands-on;
- transparent about product stage and roadmap direction.

## How The Builder should not feel

The Builder is not a childish mascot. The Builder should not feel:

- cartoonish;
- anime-styled;
- childish;
- gamer/RGB-heavy;
- hype-driven;
- like a manufacturer representative;
- like an official certification badge;
- like a payment, checkout or licensing symbol;
- like proof that the app is launched or production-ready.

## Legal and product safety rules

The Builder should not imply manufacturer partnership, certification, endorsement, sponsorship, review or official connection. The character should not appear on legal pages, privacy pages, terms pages, refund policy pages, license terms pages, cookie consent UI or manufacturer disclaimer pages.

The Builder should not be used to claim or imply that:

- a public download is available;
- checkout is active;
- paid plans are active;
- paid catalogs are active;
- license activation is active;
- an account system is active;
- mobile apps, cloud sync or printer integrations are active;
- crowdfunding pledges are active;
- public early access is active;
- manufacturer-approved or certified catalogs exist;
- printer issues can be guaranteed against.

Use cautious language around The Builder and NozzleNote: currently in development, being built, planned, future, roadmap, product direction, release preparation and not available yet.
